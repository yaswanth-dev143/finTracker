# FinTracker — Agent Guide

## Commands

```sh
# Backend (Hono + TypeScript + sql.js)
cd backend && npm run dev     # tsx watch, port 3001, auto-migrates DB
cd backend && npm run build   # tsc

# Frontend (React 18 + Vite + Tailwind v4 + shadcn v4)
cd frontend && npm run dev    # Vite dev, port 5173, proxy /api → :3001
cd frontend && npm run build  # tsc -b && vite build (do both)
```

Verify: `tsc --noEmit` then `vite build`. Always run both.

## Architecture

**Backend** (monorepo `backend/`):
- Layered: `app.ts → routes → validators → controllers → services → repositories → libs/db.ts + libs/query.ts`
- Singleton SQLite via sql.js (pure JS, no native deps). DB at `data/finance.db`, auto-created + migrated on startup.
- **CRITICAL**: `execute()` in `libs/query.ts` calls `last_insert_rowid()` **before** `saveDb()` — db.export() resets it.
- Income transactions increment `months.total_income` automatically.
- `app.ts` mounts `/api/months`, `/api/groups`, `/api/categories`, `/api/transactions`.

**Frontend** (monorepo `frontend/`):
- Pages own state (`useState`/`useEffect`/API calls) — NOT stateless.
- `TransactionModal` is the only stateless reusable component (pure props).
- UI via shadcn v4 base-nova style (uses `@base-ui/react`, NOT Radix UI). Components in `src/components/ui/`.
- Import alias `@/` → `./src/` (configured in `tsconfig.json` + `vite.config.ts`).
- `cn()` utility at `src/lib/utils.ts` (clsx + tailwind-merge).
- Tailwind CSS v4 — config is CSS-based via `@theme inline` in `src/index.css`. No `tailwind.config.js`.
- Dark mode via `.dark` class on `<html>`, toggled in `Layout.tsx`, persisted to `localStorage`.
- Vite proxy: `/api` → `http://localhost:3001`.

## Key Conventions

- No test framework installed (backend or frontend).
- Backend imports use `.js` extension in source (Hono ESM convention).
- `noUnusedLocals: false`, `noUnusedParameters: false` in frontend tsconfig.
- shadcn components are generated by `npx shadcn@latest add <name>` — do NOT hand-edit unless necessary.

## Data Model (backend)

```
months → budget_groups → categories → transactions
```
Monetary calculations (remaining, utilization %, savings, total_income) are server-side only.
